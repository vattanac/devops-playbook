Kubernetes

Understanding Services

How pods discover each other, communicate inside a cluster, and expose applications to the outside world.

01 — Foundation

Pods: the smallest deployable units

Everything you run on Kubernetes lives inside a pod. Pods are ephemeral — they can be created and killed at any time to match the desired cluster state declared by deployments.

Each pod gets its own IP address — useful, but these IPs change constantly as pods come and go. A front-end service can't hardcode the IP of a back-end pod that might be replaced at any moment.

💡

This is the core problem Services solve: stable discovery for ephemeral, ever-changing pods.

02 — Solution

Services: stable abstraction over pods

A Service gets its own stable IP and DNS name. It uses label selectors to find the right pods in a large pool, and routes traffic to them automatically — even as pods are replaced.

The service IP stays the same. The frontend doesn't need to know which pod handles the request — or that pod-c just got replaced.

03 — Selection

How services find pods: labels

Pods carry labels — key/value pairs. A service uses a selector to watch for pods matching those labels, automatically updating its routing table as pods appear and disappear.

04 — Types

Three types of services

ClusterIP
internal only

Default type — internal communication

Creates a virtual IP accessible only within the cluster. Used when front-end, back-end, and database pods need to talk to each other. No external traffic reaches this service directly.

NodePort
external access

Expose on a predefined port across all nodes

Opens a specific port on every node in the cluster. External traffic hitting NodeIP:NodePort gets forwarded to the service. Good for development and non-cloud environments.

LoadBalancer
cloud providers

Provision a cloud load balancer automatically

Only supported by cloud providers (AWS, GCP, Azure). Extends NodePort by automatically invoking the cloud's load balancer to distribute traffic to the exposed ports on all cluster nodes.
